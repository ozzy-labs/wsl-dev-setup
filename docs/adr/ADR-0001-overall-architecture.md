# ADR-0001: 全体アーキテクチャ（Bashベース）

## ステータス

承認済み

## コンテキスト

プロジェクトの初期セットアップ（ブートストラップ）において、ターゲット環境に依存せず、かつ最小限の手順で実行可能なシステムが必要です。

## 決定

コアとなるセットアップロジックは Bash スクリプトで実装します。

- `install.sh`: エントリポイント。リモート（curl | bash）実行に対応。
- `scripts/setup-local-*.sh`: OS 別のセットアップ詳細。

## 理由

1. **ポータビリティ**: ほとんどの Linux ディストリビューションおよび macOS に標準でインストールされています。
2. **依存の最小化**: ランタイム（Node.js, Python 等）をインストールする前に実行する必要があるため、それら自身に依存することはできません。
3. **Curl-ability**: 単一のシェルスクリプトをパイプに渡すだけでセットアップを開始できる利便性を重視しました。

## 影響

- スクリプトの品質管理のため、`shellcheck` および `shfmt` による自動検査を必須とします。
- 複雑なロジックは避け、各スクリプトの役割を明確に分離します。
